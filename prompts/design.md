# 설계 문서에 반드시 포함할 것
1. `## 접근 방식` — 어떤 방식으로 풀지, 핵심 결정과 그 이유
2. `## 변경/추가 파일` — 건드릴/만들 파일 경로 목록 (각 파일이 무슨 역할인지)
3. `## 인터페이스/시그니처` — 공개 함수/클래스의 이름·인자·반환·예외
4. `## 엣지케이스 처리` — 경계값·오류 입력을 어떻게 다룰지

# 제약
- 레포의 기존 구조/관례를 먼저 파악하고 그에 맞춰 설계하라.
- 코드를 작성·수정하지 마라. 오직 설계 문서 하나만 생성한다.
- 명세의 수용 기준을 모두 충족하는 설계여야 한다.
