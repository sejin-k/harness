# 개요

장바구니에 담긴 항목 목록을 입력받아 총액을 계산하는 기능을 `app` 패키지에 제공한다. 각 항목은 단가(`price`)와 수량(`qty`)을 가지며, 합계가 일정 기준 이상이면 할인을 적용한 최종 금액을 반환한다. 결제 금액 산정의 공통 로직을 한 곳에 모아 재사용성과 검증 가능성을 확보하기 위함이다.

## 범위

**이번 작업에서 다룰 것**
- `app` 패키지에 장바구니 총액 계산 함수 1개 제공
- 항목별 `price * qty`의 합계 계산
- 합계가 100000 이상일 때 5% 할인 적용 후 최종 금액 반환
- `price` 또는 `qty`가 음수일 경우 `ValueError` 발생

**이번 작업에서 다루지 않을 것**
- 통화/환율, 세금(VAT), 배송비 계산
- 쿠폰/프로모션 코드 등 5% 정액 할인 외의 할인 정책
- 영속화(DB 저장), API 엔드포인트, UI
- 동시성/트랜잭션 처리

## 함수 시그니처(제안)

```python
# app 패키지 내
def calculate_cart_total(items: list[dict]) -> float:
    ...
```

- 입력: `items` — 각 원소가 `{"price": <number>, "qty": <number>}` 형태인 리스트
- 출력: 할인 규칙이 적용된 최종 금액(숫자)

## 수용 기준 / Acceptance Criteria

- [ ] `app` 패키지에서 장바구니 총액 계산 함수를 import하여 호출할 수 있다.
- [ ] 빈 리스트 `[]`를 입력하면 `0`을 반환한다.
- [ ] 항목들의 합계가 100000 미만이면 할인 없이 `sum(price * qty)`를 그대로 반환한다. 예) `[{"price": 1000, "qty": 2}, {"price": 500, "qty": 3}]` → `3500`.
- [ ] 합계가 정확히 100000일 때 5% 할인이 적용되어 `95000`을 반환한다(경계값, 100000 이상 포함).
- [ ] 합계가 100000을 초과하면 5% 할인이 적용된다. 예) 합계 `200000` → `190000`.
- [ ] 어떤 항목의 `price`가 음수이면 `ValueError`를 발생시킨다.
- [ ] 어떤 항목의 `qty`가 음수이면 `ValueError`를 발생시킨다.
- [ ] `price` 또는 `qty`가 `0`인 경우는 정상 처리되며 해당 항목은 합계에 `0`을 기여한다.

## 영향 받는 파일/모듈

- `app/` 패키지: 총액 계산 함수가 추가될 모듈(예: `app/cart.py` 또는 `app/__init__.py`에서 노출)
- 테스트: 위 수용 기준을 검증하는 단위 테스트(예: `tests/test_cart.py`)
