# 변경 요약 (diff 발췌)
```diff
{diff}
```
- 전체 변경은 `git diff {base}..HEAD` 로, 커밋 이력은 `git log --oneline` 로 확인하라.
- 테스트는 `{test_cmd}` 로 직접 실행해 검증할 수 있다.

# 검토 관점
1. 명세의 **수용 기준을 실제로 모두 충족**하는가
2. 설계를 따르는가 (벗어났다면 정당한가)
3. 정확성·경계값·오류 처리에 결함이 없는가
4. 테스트가 수용 기준을 실질적으로 검증하는가 (형식만 통과 아님)
5. 명백한 버그·보안 문제·디버그 잔재가 없는가

# 제약
- 너는 **코드를 수정하지 않는다(읽기 전용)**. 오직 검토만 한다.
- 사소하지 않은 문제가 하나라도 있으면 `CHANGES_REQUESTED` 다.
- 배포 가능한 수준일 때만 `PASS` 다.
