    {spec_path}

# 명세 문서에 반드시 포함할 것
1. `# 개요` — 무엇을, 왜 만드는지 1~3문장
2. `## 범위` — 이번 작업에서 다룰 것 / 다루지 않을 것
3. `## 수용 기준` (영문 제목 `## Acceptance Criteria` 도 함께 표기)
   - 체크 가능한 항목들을 `- [ ]` 체크박스 목록으로 작성
   - 각 항목은 테스트로 검증 가능해야 한다 (모호한 표현 금지)
4. `## 영향 받는 파일/모듈` — 예상되는 변경 대상

# 제약
- 코드를 작성하거나 수정하지 마라. 오직 명세 파일 하나만 생성한다.
- 수용 기준은 최소 3개 이상, 구체적이고 검증 가능하게.
