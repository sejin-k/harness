# 개요

`app` 패키지에 정수 리스트를 입력받아 짝수 원소만 골라 그 합을 반환하는 함수를 구현한다. 데이터 집계 시 짝수 항목만 합산하는 공통 유틸리티가 필요하여 만든다.

## 범위

**다룰 것**
- `app` 패키지에 짝수 합산 함수 1개 구현 (예: `app/even_sum.py` 의 `sum_even` 함수)
- 정수 리스트 입력 → 짝수 원소들의 합(정수) 반환
- 빈 리스트 처리
- 정수가 아닌 원소에 대한 입력 검증 및 예외 처리

**다루지 않을 것**
- CLI / API / UI 등 외부 인터페이스
- 부동소수점·문자열 등 정수 외 타입을 합산 대상으로 포함하는 동작
- 성능 최적화, 병렬 처리, 스트리밍 입력
- 음수/0의 짝수 판정 외의 별도 비즈니스 규칙

## 수용 기준 (Acceptance Criteria)

- [ ] `app` 패키지에서 함수를 import 할 수 있다 (예: `from app.even_sum import sum_even`).
- [ ] `[1, 2, 3, 4, 5, 6]` 입력 시 `12`(2+4+6)를 반환한다.
- [ ] `[1, 3, 5]` 처럼 짝수가 하나도 없으면 `0`을 반환한다.
- [ ] 빈 리스트 `[]` 입력 시 `0`을 반환한다.
- [ ] `0`과 음의 짝수를 짝수로 취급한다: `[-2, -1, 0, 1]` 입력 시 `-2`를 반환한다.
- [ ] 원소 중 정수가 아닌 값(예: `"a"`, `2.5`, `None`)이 하나라도 있으면 `TypeError`를 발생시킨다 (예: `[2, "a"]`).
- [ ] `bool`은 정수의 하위 타입이지만 정수가 아닌 입력으로 간주하여 `TypeError`를 발생시킨다 (예: `[2, True]`).
- [ ] 입력을 변경하지 않는다(전달한 리스트가 호출 후에도 그대로 유지된다).

## 영향 받는 파일/모듈

- `app/even_sum.py` — `sum_even` 함수 신규 구현 (신규 파일)
- `app/__init__.py` — 필요 시 export 추가 (기존 파일)
- `tests/test_even_sum.py` — 수용 기준 검증용 테스트 (신규, 구현 단계에서 작성)
